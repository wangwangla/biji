## 版本控制
```
    本地控制系统、集中是版本控制系统、分布式版本控制系统。

    本地版本控制系统
        在自己电脑上一个文件修改了三次，可以回到任意修改的问题，简单，单一。所谓本地就是版本存储在本地
    集中式的版本控制
        将每个版本放在一个集中的地方，认证的用户都可以发布自己的版本上来。但是有一个问题，加入有一天机器挂掉了，就会将所有的版本丢失掉。如果网络不好的时候，会造成效率比较低下。
    分布式版本控制
        多少个开发者，就会有多少个版本

    ==============================================
```

```

    将一=个空的目录变为一个仓库

    将版本放入到暂存区git add indexXX

    .git就时暂存区和仓库的位置

## 配置用户信息
```
    git config --global user.name 'kw';
    git config --global user.email ''
    git config --list 查看自己的信息

    git add -A 将所有文件放入到缓存区中
    git add *
    git add ./

    撤销
    git checkout 文件名【将暂存区中的数据撤销】

    将暂存区中的数据放入到仓库【生成一个暂存点】
    git commit -m ""
    git commit 回车然后在进行输入

    查看仓库有哪些版本
    git log 有哪些版本
    回到历史版本
    git reset --hard XXXXXXXXXXXXXXXXXXXXX

```
## 分支笔记
```
    创建仓库
    git init
    git add *
    master至少要提交一次
    git commit -m "init"

    创建一个新的分支
    git branch 分支名称

    查看分支
    git branch

    切换分支
    git checkout xx

    返回之前的分支，将当前的先保存一下，然后在提交

    git checkout xx

```

## 总结一下
```
    git init初始化仓库
    git add -A 将代码提交一次 
    git commit -m ""

    git branch cart
    切换到cart
    git checkout cart
    git add -A
    git commit -m ""

    然后合并代码
    git merge master 在cart的时候执行就是将master合并到cart 


    删除分支，必须在其他分支上
    git branch -d fenzhi 一般在有未提交的代码时，删除是有风险的，是不允许删除掉的
    git branch -D fenzhi【强制】

```
##  多人协同开发
```
     将本地仓库中的代码放入到远程仓库中
     git push 远程仓库 master【将代码推到远程仓库】
     git pull 远程仓库 master【将代码拉取下来】

     共享仓库以xx.git结尾
     必须是一个裸仓库 git init --bare


```
## 创建一个远程的裸仓库
```
    创建一个以.git结尾的仓库
    mkdir remoteres.git
    cd remoteres.git
    创建一个裸仓库
    git init --bare
    创建结束

    git push ss@192.XX.XX.XX:xx/xx/xx.git master
    就将代码上传到了仓库中
    
    验证方法方法，将其删除，重新拉取
    git pull ss@192.XX.XX.XX:xx/xx/xx.git master
